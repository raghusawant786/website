# Repository Map

This is a Docusaurus portfolio website for Raghunath Sawant. It presents a QA automation profile, docs-style resume sections, certifications, blog content, and a homepage with interactive React sections.

## Tech Stack

- Framework: Docusaurus 3.x
- UI: React 18
- Content: Markdown and MDX
- Styling: Docusaurus theme CSS plus CSS Modules
- Package manager: Yarn
- Deployment target: GitHub Pages

## Main Commands

```bash
yarn start
yarn build
yarn serve
yarn deploy
```

The configured site URL is `https://raghunahtsawant.github.io` with `baseUrl: '/website/'`.

## Important Root Files

- `package.json` - scripts and dependencies.
- `yarn.lock` - resolved dependency tree.
- `docusaurus.config.js` - site metadata, navbar, footer, plugins, presets, analytics script, theme config.
- `sidebars.js` - docs sidebar. Currently autogenerated from `docs/`.
- `README.md` - project setup notes.
- `ANALYTICS_SETUP.md` - Google Analytics, SEO, sitemap, accessibility, and performance notes.
- `IMPROVEMENTS_SUMMARY.md` - historical summary of accessibility, image, analytics, and component improvements.
- `.github/skills/portfolio-project/SKILL.md` - local project conventions for future Codex work.

## Homepage Composition

`src/pages/index.js` renders:

- `HomepageHeader` - hero section, CTA links, resume download, social links.
- `ProfessionalStats` - animated counters for experience, coverage, tools, team, defect reduction, certifications.
- `TechSkillsShowcase` - filterable skills grid.
- `ProjectShowcase` - filterable QA project/case-study cards.
- `ResumePreview` - resume highlights plus preview/download/contact actions.
- `Testimonials` - testimonial section.
- `ExperienceTimeline` - career timeline.
- `ScrollProgress` - fixed top reading/progress indicator.

Each major homepage section is wrapped in `ErrorBoundary`.

## Styling Rules

- Site-wide styles live in `src/css/custom.css`.
- Component styles live beside components as CSS Modules, usually `styles.module.css`.
- The local skill file says to prefer CSS Modules over inline styles, except small dynamic color values that are already part of the current pattern.
- Responsive breakpoints commonly use `768px` and `480px`.

## Current Docusaurus Notes

`package.json` currently declares Docusaurus `^3.10.1`, React `^18`, and MDX `^3`. This means MDX syntax is stricter than Docusaurus 2-era projects, and Node 18+ is expected.

The Docusaurus config still uses CommonJS and imports Prism themes from:

```js
require('prism-react-renderer/themes/github')
require('prism-react-renderer/themes/dracula')
```

That should be checked during builds because Docusaurus 3 and newer Prism packages changed some import paths across versions.

## Quick Risk Checklist

- Run `yarn build` after changing Docusaurus config, package versions, MDX files, routes, or imports.
- Check image paths under `static/img/`; some certification JSON entries use `.png` while current assets include several `.svg` files.
- Replace placeholder Google Analytics ID `G-XXXXXXX` before expecting analytics data.
- Confirm `organizationName`, `url`, and repository links are correct before deploy.
- Keep generated build output out of source edits unless deployment specifically needs it.
